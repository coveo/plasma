# About Content

"Content" is the information in the UX working alongside the visual design.
It includes any text, images or videos in the Coveo Administration Console interface that help shape how users navigate the product.

## Who is this for?

This style guide is intended for anyone who contributes to the writing, editing, or reviewing of UX content.

This is often:

- Designers
- Developers
- Technical writers

## Glossary

- **Content guidelines**: Rules established for content to keep the UX clear, consistent, and on brand.
- **Long-form copy**: Extended text that provides context, detail, or instruction in several paragraphs.
  Long-form doesn't appear in the Administration Console, but is typical for product documentation and Level Up+ content.
- **Microcopy**: The broader category of short, functional text that guides users at decision points, including UI copy and supporting text like tooltips, placeholder text, error messages, and helper text.
- **Short-form copy**: Brief, scannable text designed to communicate a single idea and reduce confusion.
  Short-form can be anywhere from a few words to a couple of paragraphs.
  This includes page titles, descriptive text for panels and pages, confirmation text, and banner text.
- **UI copy**: The text rendered as part of a UI component itself, such as button labels, menu items, field labels, and status messages.
- **UX copy**: Also called _UX writing_. The language within an interface that guides users throughout their whole experience.
