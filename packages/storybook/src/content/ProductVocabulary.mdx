# Product Names & Vocabulary

This page defines the correct spelling and capitalization for Coveo product and feature names.
Vocabulary isn't standardized across all Coveo sources or documentation—always refer to this list when naming a product, feature, or tool name in the UI.
Terms are categorized by capitalization style: [sentence case](#sentence-case) and [Title Case](#title-case), appearing alphabetically.

<Unstyled>
    <Alert.Advice title="Term not listed?" mb="md">
        If a product or feature name isn't listed here, look up whether it's a proper noun or has an established capitalization convention and use the most widely-used form.
        If you're unsure, submit a <a href="https://coveord.atlassian.net/jira/software/c/projects/DS/form/600">request</a> and select <strong>Content &amp; Copywriting</strong> as the "Type".
    </Alert.Advice>
</Unstyled>

## Sentence case

This section lists the terminology that should be in sentence case when writing UI body text, labels, and descriptions.

- agent
- analytics (Formerly _usage analytics_. Use when referring to data or events, e.g., "the analytics data." Only when distinguishing from a third-party product, use _Coveo Analytics_)
- answer generation
- API key (_API_ is always capitalized)
- catalog
- catalog configuration
- condition
- conversational AI (_AI_ is always capitalized)
- dimension
- evaluation
- extension
- field
- filter rule
- group
- in-product experience
- indexing pipeline extension
- listing page
- machine learning (when referring to the technology in general. Only when distinguishing from a third-party product, use _Coveo Machine Learning_)
- member
- model (when referring to a model generically, e.g., "associate the model with a query pipeline." When naming the type, use the specific model name in [Coveo products and ML models](#coveo-products-and-models))
- named filter
- notification
- passage retrieval (when referring to the capability or queries, e.g., "passage retrieval queries per second." When naming the model type, use _Passage Retrieval_)

- query pipeline

- query pipeline setting
- ranking expression rule

- report

- security identities
- security identity provider
- security provider
- source

- statement group
- stop word rule
- storefront association
- thesaurus rule
