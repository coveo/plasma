# Accordion

Collapsible content panels with an additional disabled control variant.

## Overview

Use an Accordion to organize related content into collapsible sections.

It works well when vertical space is limited and not all content needs to be visible at once — for example, a settings panel where users typically only need one section at a time.

## Props

This component has no additional props beyond the Mantine base. Refer to the [Mantine documentation](https://mantine.dev) for all available props.

## Sub-components

Plasma wraps Mantine's Accordion sub-components with pre-configured defaults. Use these instead of setting props manually.

| Sub-component               | Purpose                                                   |
| --------------------------- | --------------------------------------------------------- |
| `Accordion.Item`            | Wrapper for a single accordion entry                      |
| `Accordion.Control`         | The clickable header that expands or collapses the panel  |
| `Accordion.ControlDisabled` | A header with no chevron and disabled pointer events      |
| `Accordion.Panel`           | The content area revealed when an item is expanded        |

Use `Accordion.ControlDisabled` for items that are always expanded and can't be collapsed by the user.
It removes the chevron icon and disables pointer events so the control doesn't suggest interactivity.

## Usage

```tsx
import {Accordion} from '@coveord/plasma-mantine';

function Example() {
    return (
        <Accordion defaultValue="details">
            <Accordion.Item value="details">
                <Accordion.Control>Project details</Accordion.Control>
                <Accordion.Panel>Configure the project name, description, and visibility settings.</Accordion.Panel>
            </Accordion.Item>

            <Accordion.Item value="readonly">
                <Accordion.ControlDisabled>Always visible section</Accordion.ControlDisabled>
                <Accordion.Panel>This content is always visible and cannot be collapsed.</Accordion.Panel>
            </Accordion.Item>
        </Accordion>
    );
}
```

The first item uses a standard `Accordion.Control` — the user can expand and collapse it.

The second item uses `Accordion.ControlDisabled` — the content is always visible and the control gives no indication it can be toggled.

## Content guidance

**Control labels (`Accordion.Control`):**

- Use sentence case
- Keep labels short — ideally 1–4 words
- Name the content inside, not the action: "Project details" not "View project details"
- Don't end with a colon or punctuation

**Panel content (`Accordion.Panel`):**

- Write as full sentences when the content is descriptive; use fragments only for lists or form fields
- Add a period after full sentences; don't add a period after fragments
- Keep content focused — if a panel needs multiple headings, consider a different layout

**Ordering (`Accordion.Item`):**

- Put the most frequently used or important section first
- Group related items together
- Don't mix always-visible items (`Accordion.ControlDisabled`) with collapsible ones unless there's a clear reason — if you do, put the always-visible items last
