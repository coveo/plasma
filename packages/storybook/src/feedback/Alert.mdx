# Alert

Alert callout for contextual information, advice, warnings, critical errors, and success states.

## Overview

Use an Alert to surface an in-page message that needs the user's attention — a tip, a caution, an error, or a confirmation that something succeeded. Alerts are static and contextual; they appear inline near the content they relate to, not as overlays or toasts.

Choose the variant that matches the nature of the message so the visual weight matches the urgency. Use `Alert.Critical` only for genuine errors — overusing high-urgency variants reduces their effectiveness.

## Content guidance

**`title`:**

- Use sentence case
- Keep to 1–4 words
- Name the situation, not the severity: "Unsaved changes" not "Warning"
- Don't end with punctuation

**Alert body:**

- Write as one or two full sentences ending with a period
- Lead with what happened or what the user needs to know, then add context or next steps
- Keep to 25 words or fewer
- Use plain language — avoid jargon and technical terms the user may not know

**Variant selection:**

- Use `Alert.Information` for neutral context that isn't time-sensitive
- Use `Alert.Advice` for optional tips that improve the experience
- Use `Alert.Warning` when ignoring the message could cause data loss or errors
- Use `Alert.Critical` for failures that block the user or require immediate action
- Use `Alert.Success` only after a user-initiated action completes — don't use it for passive status updates
