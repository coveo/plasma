# ActionIcon

Icon-only button that can trigger actions and can show a disabled-state tooltip.

## Overview

Use ActionIcon when an action is best represented by an icon alone — for example, delete, close, or edit controls in compact layouts where a full button label would clutter the UI.

It extends Mantine's ActionIcon with support for async click handlers and a tooltip that appears when the button is disabled, letting you explain why the action isn't available.

## Props

> Extends `MantineActionIconProps`. Only Plasma-specific props are listed below. Refer to the [Mantine documentation](https://mantine.dev) for inherited props.

**`onClick`** `ClickHandler<HTMLButtonElement>` · optional

Handler executed on click. Supports standard, async, and parameterless handlers. When you pass an async handler, the button automatically shows a loading state while the promise resolves — you don't need to manage that state yourself.

**`disabledTooltip`** `string` · optional

Tooltip message shown when the button has `disabled` set to `true`. Use this to explain why the action isn't available rather than leaving the user to guess.

**`disabledTooltipProps`** `Omit<TooltipProps, 'disabled' | 'label' | 'children'>` · optional

Additional props passed to the disabled-state tooltip. Use this when you need to adjust positioning, delay, or other tooltip behavior beyond the default.

## Usage

```tsx
import {ActionIcon} from '@coveord/plasma-mantine';
import {IconTrash} from '@coveord/plasma-react-icons';

const removeItem = () => {};
const deleteItem = async () => {};

// Common pattern: use a sub-component for the intended emphasis.
<ActionIcon.Quaternary onClick={removeItem}>
    <IconTrash aria-label="Remove" />
</ActionIcon.Quaternary>;

// Async handlers automatically show a loading state while pending.
<ActionIcon.DestructivePrimary onClick={deleteItem}>
    <IconTrash aria-label="Delete" />
</ActionIcon.DestructivePrimary>;

// Disabled actions MAY explain why they are unavailable.
<ActionIcon.Primary disabled disabledTooltip="You do not have permission to remove this item">
    <IconTrash aria-label="Remove" />
</ActionIcon.Primary>;
```

Choose the sub-component that matches the emphasis level of the action. Use `Destructive` variants for actions that permanently remove or alter data. Pass an async function to `onClick` when the action triggers a network request — the loading state is handled for you.

## Content guidance

**Icon `aria-label`:**

- Write a short verb phrase that describes the action: "Delete", "Remove", "Close", "Edit"
- Use sentence case
- Don't repeat the surrounding context — "Delete" not "Delete this item"
- Don't use "icon" or "button" in the label

**`disabledTooltip`:**

- Explain why the action isn't available, not just that it's unavailable
- Start with the reason: "You don't have permission to remove this item" not "This action is disabled"
- Keep to 20 words or fewer
- Write as a full sentence ending with a period
